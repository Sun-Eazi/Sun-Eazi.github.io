## 1. Abstract
This report explores the implementation of a vision-based AI system to classify urban transit vehicles in Dar es Salaam, Tanzania. The focus is on Bajaj, Daladala, and Bodaboda—the backbone of the city's transport ecosystem.

## 2. Problem Statement
Dar es Salaam faces significant traffic congestion. Manual classification of vehicles for traffic flow analysis is labor-intensive and prone to error. An automated system is required to provide real-time data to urban planners.

## 3. Methodology: OOAD Approach
To ensure the project reaches "World-Class" status, we adopted Object-Oriented Analysis and Design (OOAD):

### 3.1 Abstraction
We abstracted the "Vehicle" into a domain entity that contains properties like `class`, `color`, and `confidence`, independent of the AI model used.

### 3.2 Polymorphism
The system supports multiple "Classifiers". While we currently use `GeminiClassifier`, the architecture allows for `YOLOClassifier` or `TensorFlowClassifier` to be plugged in without changing the core business logic.

### 3.3 Dependency Inversion
Higher-level business logic (the classification service) does not depend on low-level database details. Both depend on persistent interfaces.
